注文・進捗管理UIプロトタイプ

注文を受けてから提供までの進捗を管理するUIプロトタイプ

Web UI DB Design

概要

飲食店などの運用を想定し、注文の受付から進捗管理までを 画面上で分かりやすく把握できるUIプロトタイプを制作しました。

注文データは画面から消えても保持される設計を前提とし、 将来的な売上管理・在庫管理への拡張も考慮しています。

画面イメージ

注文一覧を俯瞰できるメイン画面

1注文ごとの注文明細表示

進捗状態を切り替えて管理

複数画面運用を想定したUI構成

主な機能

  • 注文単位でのデータ管理
  • 注文明細（商品・数量）の保持
  • 進捗状態の管理（例：受付／調理中／提供）
  • 画面から消えても注文データは削除しない設計

設計のポイント

  • 「1会計＝1注文」というシンプルなデータモデル
  • 進捗は注文単位で管理
  • UIとデータ設計を分離し、将来拡張しやすく設計

使用技術

  • Frontend：React（HTML / CSS / JavaScript）
  • Backend想定：PHP / DB（設計段階）
  • データ設計：注文・注文明細テーブル構成

工夫した点

  • 現場での視認性を重視したシンプルなUI
  • 画面数が増えても混乱しにくい構成
  • 実運用を想定したデータ保持方針
